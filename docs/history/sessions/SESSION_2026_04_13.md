### 2. Refactor Match→Contrato consolidado
**Problema**: El módulo "Contratos > Candidatos Aprobados" (`/app/contracts/matched-candidates`) era una vista paralela duplicada al tab "Aceptadas" de Solicitudes. Confundía al arrendador: *"si ya acepté el match, ¿por qué tengo que ir a otro módulo?"*.

**Bug raíz descubierto**: `matching/models.py:accept_match()` solo cambiaba status, **no creaba el `Contract`** → el endpoint `matched-candidates` devolvía candidato sin `contract_info`, y la UI mostraba "no se encontró el contrato".

**Fix backend** (`matching/models.py:179-222`):
- `accept_match()` ahora actualiza `workflow_stage=1, workflow_status='visit_pending'`
- Nuevo método `_ensure_contract_exists()` crea `Contract` legacy automáticamente con `contract_type='rental_urban'`, `status='draft'`, vinculado al match via `match_request` FK
- `contract_number` se autogenera en `Contract.save()` → `VH-2026-000001`

**Fix frontend**:
- `frontend/src/components/matching/MatchesDashboard.tsx`: tab "Aceptadas" para landlord ahora embebe `<MatchedCandidatesView />` (reutiliza toda la lógica de stages/modales sin duplicar código)
- Soporte query param: `?tab=accepted|rejected|cancelled` selecciona tab inicial
- `frontend/src/routes/contracts.tsx`: ruta `/matched-candidates` → `<Navigate to="/app/matches?tab=accepted" replace />`
- `frontend/src/components/contracts/ContractList.tsx`: eliminado botón "Candidatos Aprobados" + imports huérfanos (`Stack`, `PeopleIcon`)

**Validación**:
- `npx tsc --noEmit` → 0 errores
- Playwright `multi-user-contract-signing.spec.ts` → **1/1 passed (3.8 min)** · contrato final `active` · confidence 87.7%
